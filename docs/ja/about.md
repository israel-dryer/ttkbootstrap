# このプロジェクトについて

## 美しくデザインされた使いやすいスタイルを作成
ご存知の通り、ttkスタイルの作成は非常に時間がかかります。
このライブラリは、モダンな外観を簡単に実現し、25個ものスタイル設定を調整する代わりに、
アプリケーションのデザインに集中できるようにします。

## キーワードでウィジェットスタイルを設定
シンプルに。キーワードでスタイルを設定します。
従来の**success.Horizontal.TProgressbar**のような複雑なクラスではなく、
**success**というキーワードで同じ意味をすべてのウィジェットに適用できます。

多くの人がWeb開発でBootstrapを使った経験があるでしょう。
Bootstrapは、プロフェッショナルで一貫性のあるAPIを提供するCSSスタイルクラスを備え、迅速な開発を可能にします。
このプロジェクトも同様のアプローチを採用し、
ほぼすべてのttkウィジェットに事前定義スタイルを提供し、
シンプルなキーワードでカスタマイズできるようにしました。

## 実際に使用するテーマとスタイルのみを作成
使用しないものはメモリを消費すべきではありません。不要なアセットがアプリケーションを遅くします。

この問題を解決するため、ttk用のスタイリングエンジンを設計しました。
このエンジンは、スタイルとテーマをオンデマンドで構築します。
未使用のスタイルは作成されません。
これにより、テーマとスタイル設計に柔軟性が生まれ、事前読み込みされた画像ベースのウィジェットスタイルのメモリ制限に縛られなくなります。

例として、バージョン0.5では、
スケールウィジェット1つに対して288枚の画像を読み込む必要がありました。
テーマとカラーの組み合わせをすべて考慮するためです。従来のttkではこうした処理が必要でした。
しかし、バージョン1.0では、ホバー効果などを考慮しても3〜4枚の画像で済みます。
実際に使用するスタイルのみが構築されます。
